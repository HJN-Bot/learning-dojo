# SKILL — Feynman Coach (Learning Protocol)

## Purpose
Turn any topic into **understanding + practice + a reusable note**, with consistent structure.

## Inputs (what you send)
- Topic
- Why (what problem it solves)
- Current level
- Acceptance criteria for today (explain / implement / ship)
- Optional: material links or snippets

## Default flow
### Step 0 — Intake (3 clarifying questions)
1) What problem are you solving with this topic?
2) What is your current baseline?
3) What is the acceptance criteria today?

### Step 1 — Feynman Explain (3 layers)
- Kid version (<=120 words)
- Engineer version (<=200 words, include key terms)
- Boundaries / counterexamples (<=200 words)

### Step 2 — Socratic Deep Dive (5 questions)
Why / How / Tradeoff / Failure modes / Example

### Step 3 — Knowledge Check
- 3 MCQs (concept discrimination)
- 2 short answers (must re-derive in your own words)
- 1 practice task (<=20 minutes, measurable output)

### Step 4 — Publish (1-page note)
- Title
- 3 core points
- 1 analogy
- 1 worked example
- 1 common pitfall
- Next 3 actions (each <=30 min)

## Spaced repetition
- D0: complete Steps 1–3
- D2: redo quiz + fix mistakes
- D7: publish the 1-page note + add to index
